========================================
   图生图功能 - 快速测试指南
========================================

🔍 问题诊断：
您看到"模拟图"是因为浏览器缓存了旧版本的JS文件。

✅ 解决方案（3步）：

┌────────────────────────────────────┐
│  步骤1：清除浏览器缓存            │
└────────────────────────────────────┘

方法A：强制刷新（推荐）
  按 Ctrl + F5（Windows）
  或 Cmd + Shift + R（Mac）

方法B：清除缓存
  Chrome/Edge:
  1. F12 打开开发工具
  2. 右键点击刷新按钮
  3. 选择"清空缓存并硬性重新加载"

┌────────────────────────────────────┐
│  步骤2：确认服务器状态            │
└────────────────────────────────────┘

后端服务器是否运行？
  □ 是 - 继续步骤3
  □ 否 - 双击运行：scripts\启动后端服务器.bat

检查方法：
  打开：http://localhost:3000/health
  应该看到：{"status":"ok"}

┌────────────────────────────────────┐
│  步骤3：重新测试                  │
└────────────────────────────────────┘

1. 打开：http://localhost:3000/tool.html
2. 切换到"图生图"标签
3. 上传一张图片
4. 输入效果描述，例如：
   "转换为油画风格，增强色彩"
5. 点击"开始转换"

预期结果：
✅ 生成真实的AI图片（非模拟）
✅ 图片会实时从Pollinations.ai加载

========================================

🧪 快速测试（可选）

如果上述步骤仍有问题，可以使用独立测试页面：

打开：test-img2img-direct.html

这个页面直接调用Pollinations.ai，不依赖缓存。

========================================

💡 技术说明：

当前配置：
✅ img2img.js 版本已更新为 v=3
✅ 前端会自动降级到 Pollinations.ai
✅ 无需配置 Hugging Face API

工作流程：
1. 检查后端是否可用
2. 如果后端未配置HF API → 使用Pollinations
3. 生成真实图片（非模拟）

Pollinations.ai 特点：
✅ 完全免费
✅ 无需API Key
✅ 实时生成
⚠️ 不是真正的图生图（只是文字生图）
   但可以通过描述实现风格转换效果

========================================

❓ 常见问题

Q1: 还是看到"模拟图"？
A: 请务必按 Ctrl+F5 强制刷新，清除缓存

Q2: 图片加载很慢？
A: Pollinations.ai 是免费服务，首次生成需要15-30秒

Q3: 想要真正的图生图？
A: 需要配置 Hugging Face API
   查看：docs\HuggingFace配置指南.md

Q4: 生成的图片和上传的图片无关？
A: Pollinations.ai 是文字生成图片
   它不使用上传的图片，只根据文字描述生成
   要真正的图生图，需要配置 HF API
